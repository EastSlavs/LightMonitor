## 一、 产品概述 (Product Overview)

**项目定位：** 一款轻量级、免安装的 PC 硬件状态后台监控与复盘工具。

**设计初衷：** 针对高负载应用场景（如大型 3A 游戏），提供长期的硬件状态监控。现有监控方案往往带有冗余功能、注册表残留，且缺乏直观的长线历史数据可视化能力。

**解决方案：** 采用 UI 渲染与后台采集彻底分离的架构。监控期间 UI 完全休眠，后台以极低资源消耗记录数据；任务结束后，提供历史性能折线图用于瓶颈分析。

## 二、 核心特性 (Key Features)

1. **单文件便携 (Portable)：** 免安装，不写注册表。程序编译为单个 `.exe` 文件，数据与配置自动生成于存储目录，支持多端便携使用。
2. **极低后台开销 (Low Overhead)：** 程序最小化至托盘时，全面挂起 UI 渲染引擎。后台采集线程以低频（默认 1Hz）调用底层硬件接口，将性能损耗降至最低。
3. **高性能图表 (High-Performance UI)：** 采用底层基于 C++ 的 `pyqtgraph` 框架，支持海量数据点的高帧率滑动与交互。
4. **本地轻量级存储 (Local Storage)：** 采用开启 WAL（预写日志）模式的 SQLite 数据库存储历史数据。支持自定义数据保留周期，过期自动清理机制。

## 三、 技术栈 (Technology Stack)

- **语言：** Python 3.x
- **GUI 框架：** PyQt5
- **数据可视化：** pyqtgraph
- **硬件接口：** - `psutil` (跨平台系统资源监控：CPU、内存)
  - `pynvml` (NVIDIA 官方管理库：GPU 占用、显存、温度)
- **数据存储：** SQLite3
- **构建工具：** PyInstaller

## 四、 架构设计 (Architecture Design)

系统采用双线程 MVC 架构，确保数据采集与界面渲染严格解耦：

### 1. 采集线程 (Worker Thread)

- **职责：** 独立于主线程在后台运行。
- **机制：** 定时调用硬件接口读取性能指标 (CPU、GPU、内存、温度) -> 执行 SQL 语句写入本地 `.db` 文件。
- **特性：** 运行过程不涉及任何 UI 操作，规避主线程阻塞。

### 2. UI 主线程 (Main Thread)

- **职责：** 管理窗口生命周期与图表渲染。
- **机制：**
  - **静默状态：** 窗口隐藏，彻底释放图形渲染资源。
  - **唤醒状态：** 接收托盘唤醒指令，从 SQLite 查询指定时间窗口的历史记录，并初始化 `pyqtgraph` 绘图。
  - **活动更新：** 窗口显示期间，监听采集线程的信号槽，执行图表的增量刷新。

## 五、 开发路线图 (Development Roadmap)

- **Phase 1：接口联调 (已完成)** 打通硬件底层接口，实现 CPU、内存、GPU 及温度数据的读取与验证。
- **Phase 2：数据持久化** 引入 SQLite3 并开启 WAL 模式，处理动态路径识别，实现自动建表与定时写入逻辑。
- **Phase 3：UI 骨架搭建** 基于 PyQt5 开发无图表的框架窗体，实现托盘接管、窗口隐显控制与安全退出机制。
- **Phase 4：数据可视化** 集成 `pyqtgraph` 组件，对接数据库实现历史数据拉取与动态折线图渲染，解决高频重绘的性能问题。
- **Phase 5：配置与发布** 增加 JSON 配置系统（支持自定义保留天数），完善数据库生命周期管理（过期清理），配置 PyInstaller 输出单文件发行版。
